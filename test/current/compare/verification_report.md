## 2. 关键错误详解

### 错误 1：栅距 pin pitch 错误（影响所有变体）

**位置**：所有 6 个变体输入卡的几何定义区
```serpent
surf s1 sqc 0.0 0.0 0.75   % 半宽 0.75 cm → 整栅距 1.50 cm
```

**文章要求**：Table 2.3 第 4 行明确写明 *Pin Pitch = 1.26 cm*（半宽 0.63 cm）。

**影响**：
- 慢化比 V_moderator/V_fuel 由 1.34 增大到 2.32（增加 73%）
- H/HM 比由 0.159 增大到 0.275（增加 73%）
- 钍基燃料对慢化非常敏感，过度慢化使初始 kinf 虚高
- 与文章的 BOL 偏差：0.7491 − 0.7050 = **+4411 pcm**
- 与文章随燃耗的偏差：4000 – 9500 pcm（见 kinf_compare_table.csv 的 delta_MCODE_pcm 列）

**修正方法**：把 `sqc 0.0 0.0 0.75` 改为 `sqc 0.0 0.0 0.63`。

### 错误 2：pin_powden 变体的比功率错误

**位置**：`pin_powden/20000/blanket/pin_cell.sss` 和 `pin_powden/5000/blanket/pin_cell.sss`
```serpent
set powdens 0.03562    % MW/kgHM
```

**文章要求**：Table 2.3 给 *Power Density = 79.42 kW/liter of fuel*。

**反推验证**：
- 钍基毯中重元素 (HM) 数密度合计：Th-232 1.92206E+22 + U-234 1.71054E+18 + U-235 2.11441E+20 + U-238 1.87893E+21 = 2.12178E+22 (1/cm³)
- HM 质量密度：8.2304 g/cm³（占 9.3624 g/cm³ 的 87.9%）
- 比功率 = 79.42 kW/L ÷ 8.2304 kg/L = **9.6496 kW/kgHM = 9.65E-3 MW/kgHM**

**问题**：0.03562 MW/kgHM 比正确值高 3.69 倍。这意味着燃耗时间被压缩了 3.69 倍，会导致：
- 单位步长下的中子通量虚高
- 同名 "MWd/kgHM" 实际对应的中子注量率与文章不同
- 不能直接与文章/CASMO-4/MCODE 对比

**修正方法**：改为 `set powdens 9.65E-3`。

### 错误 3：pin_power 变体的单 pin 功率错误

**位置**：`pin_power/200000/blanket/pin_cell.sss` 和 `pin_power/5000/blanket/pin_cell.sss`
```serpent
set power 178.695    % W
```

**正确的单 pin 功率**：
- 单 pin 燃料体积（每 1 cm 高）= π × 0.4646² = 0.6782 cm³
- 文章 79.42 kW/L = 79.42 W/cm³（按**燃料**体积计）
- 单 pin 功率（每 cm 高）= 79.42 × 0.6782 = **53.86 W**

**问题**：178.695 W 是把功率密度应用到整个栅元（1.5×1.5 cm 截面）体积上得到的：
- 79.42 W/cm³ × 2.25 cm² = 178.7 W（这是整个 cell 的功率，不是单 pin 燃料的功率）
- 偏差倍数 178.7 / 53.86 = 3.32× — 这恰好等于 1.5² / (π × 0.4646²)，即"错误 1 + 错误 3"的复合效果

**修正方法**：改为 `set power 53.86`（或最好统一用 `set powdens 9.65E-3`，更直观）。

## 4. 修复步骤

### 推荐方案
1. **所有变体**：将 `surf s1 sqc 0.0 0.0 0.75` 改为 `surf s1 sqc 0.0 0.0 0.63`
2. **pin_powden 变体**：将 `set powdens 0.03562` 改为 `set powdens 9.65E-3`
3. **pin_power 变体**：将 `set power 178.695` 改为 `set power 53.86`
4. 重新运行全部 6 个变体
5. 重新执行 `python script/extract_kinf_burnup.py` 抽取数据
6. 预期结果：与文章的偏差收敛到 ±500 pcm 以内（与 CASMO-4↔MCODE 自身的偏差量级相当）

### 修复后可立即验证的判断
- BOL kinf 应在 0.700 – 0.710 区间
- 燃耗末 (120 MWd/kgHM) kinf 应在 0.71 – 0.73 区间
- 全程 ΔMCODE 绝对值 < 1500 pcm

## 5. 备注：文章自身的一处小不一致

文章第 45 页（表 4. 燃料棒数据）写 *Cladding outer diameter = 9.5 mm*，
第 46 页（表 2.3）写 *Cladding Inner Radius = 0.4728 cm* 且厚度 0.57 mm，
按后者外径应为 0.4728 × 2 + 0.057 × 2 = 1.0596 cm = 10.60 mm。
两处数字对不上；输入卡片取的是 Table 2.3 (10.6 mm OD)，这是合理的，
因为 Table 2.3 才是组件级计算用的真实参数。
建议你在论文里也以 Table 2.3 为准。

---

**结论**：输入卡片的材料定义完全正确，**但几何栅距取错（1.5 cm 应为 1.26 cm），且 powden / power 两种功率变体各自还有独立的功率设置错误**。这是当前 kinf 偏差 4000-9500 pcm 的根本原因，与 Monte Carlo 统计误差、数据库版本、燃耗算法均无关。
